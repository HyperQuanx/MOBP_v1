다양한 개발 용어 정리


정보기술(information technology : IT)

  • 전기 통신, 방송, 컴퓨팅(정보처리, 컴퓨터 네트워크, 컴퓨터 하드웨어, 컴퓨터 소프트웨어, 멀티미디어), 통신망 등 사회 기반을 형성하는 유형 및 무형의 기술 분야

인터페이스(interface)

  • 서로 다른 두 개의 시스템, 장치 사이에서 정보나 신호를 주고받는 경우의 접점이나 경계면

프로토콜(Protocol)

  • 컴퓨터 내부에서, 또는 컴퓨터 사이에서 데이터의 교환 방식을 정의하는 규칙 체계

TCP(Transmission Control Protocol)

  • 컴퓨터가 다른 컴퓨터와 데이터를 통신하는 규약

프로세스(process)

  • 컴퓨터에서 연속적으로 실행되고 있는 프로그램 스케줄링의 대상이 되는 작업(task)이라는 용어와 거의 같은 의미로 쓰임

스레드(thread)

  • 어떠한 프로그램 내에서, 특히 프로세스 내에서 실행되는 흐름의 단위

멀티스레드(Multithreading)

  • 하나의 프로세스 내에서 여러 스레드(Thread)를 생성하여 동시에 작업을 수행하는 방식

멀티프로세스(Multiprocessing)

  • 여러 개의 독립된 프로세스가 동시에 실행되면서 작업을 수행하는 방식

세션(Session)

  • 클라이언트로부터 오는 일련의 요청을 하나의 상태로 보고 그 상태를 일정하게 유지하는 기술
  • 클라이언트가 웹 서버에 접속해있는 상태가 하나의 단위


그 밖의 약어 정리

  • API(Application Programming Interface) : 프로그램들이 서로 소통할 수 있게 해주는 중간 다리 역할을 하는 도구
  • SEO(Search Engine Optimize) : 검색 엔진 최적화
  • DAO(Data Access Object) : DB의 데이터에 접근하기 위한 객체
  • DTO(Data Transfer Object) : 계층 간(Controlelr, View, Business Layer) 데이터 교환을 위한 Java Bean를 의미
  • VO(Value Object) : Read-Only 속성을 가진 값 오브젝트
